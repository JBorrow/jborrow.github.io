What your advisor probably doesn't know about python that you should

Sun, Sep 23, 2018

‘Tis the season for new PhD students to join the ranks. If you are going into any field of Physics (or any other science for that matter), there’s a good chance that you’re going to get told to use python for pretty much everything going forward. Reading data, running analysis, making figures… Welcome to your new life. Going forward, I’m going to assume you are at least somewhat familiar with the python programming language. If you’re not, I suggest checking out one of the great online courses that MIT offers.

There’s several major problems with the, now prolific, use of programming in science. Firstly, there are few opportunities for mentoring and code review - this is something you’re going to mainly have to figure out on your own. Secondly, bad practices are everywhere. The first one to get out of the way is that there is no reason whatsoever that you, as a science graduate student, should be using python2. If your advisor has some old code that they insist you use, you can probably just use 2to3 to upgrade it. If you really need a list of reasons that python3 is better than python2 (other than the fact that support for python2 is rapidly faltering), check out this great GitHub repository (my favourite new feature is f strings). Another consequence of these bad practices, among other things, is that people rarely learn how to ‘properly’ write python code. There are many definitions of the word ‘properly’ in this context, but here I’ll use it to mean extensible, readable, and clean code.

Hopefully the tips and ideas below can help you figure things out as a new graduate student and find your way in the python world.
